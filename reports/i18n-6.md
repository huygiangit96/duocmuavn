# I18N-6 - next-intl routing va messages

## Muc tieu

Cai `next-intl`, cau hinh routing song ngu `vi` / `en`, tao messages va di chuyen App Router sang segment `[locale]`.

## Dependency

Da cai trong `frontend/`:

```powershell
npm install next-intl
```

Package da them:

- `next-intl`

## File da tao / cap nhat

### I18N config

- `frontend/src/i18n/routing.ts`
  - `locales: ["vi", "en"]`
  - `defaultLocale: "vi"`
  - `localePrefix: "as-needed"`

- `frontend/src/i18n/request.ts`
  - Dung `getRequestConfig` tu `next-intl/server`
  - Load messages tu `frontend/messages/<locale>.json`
  - Fallback ve `vi` neu locale khong hop le

- `frontend/src/middleware.ts`
  - Dung `createMiddleware(routing)`
  - Matcher loai tru `api`, `_next`, `_vercel`, static files

- `frontend/next.config.ts`
  - Wrap config bang `createNextIntlPlugin("./src/i18n/request.ts")`

### App Router

Da di chuyen route hien co vao:

- `frontend/src/app/[locale]/`

Giu lai o goc `frontend/src/app/`:

- `globals.css`
- `favicon.ico`

Cap nhat:

- `frontend/src/app/[locale]/layout.tsx`
  - Import `../globals.css`
  - Validate locale theo `routing.locales`
  - Load messages bang `getMessages()`
  - Boc app bang `NextIntlClientProvider`
  - Giu `Providers` hien co cua ung dung

### Messages

Da tao:

- `frontend/messages/vi.json`
- `frontend/messages/en.json`

Hai file co day du cac namespace:

- `nav`
- `header`
- `cart`
- `footer`
- `product`
- `order`
- `auth`
- `account`
- `news`
- `docs`
- `contact`
- `common`

## Kiem tra

Da chay:

```powershell
npm run build
```

Ket qua:

- PASS
- Next.js compile thanh cong
- TypeScript PASS

Da kiem tra HTTP:

```text
http://localhost:3000    -> 200
http://localhost:3000/en -> 200
```

## Ghi chu

- Next.js 16 hien warning convention `middleware` deprecated va goi y dung `proxy`. Theo yeu cau prompt, file `frontend/src/middleware.ts` van duoc tao va hoat dong binh thuong.
- Chua thay text UI sang `useTranslations`; buoc nay moi cai dat nen tang routing/messages cho i18n frontend.
